This is not a guide to installing arch linux on your computer, but just sharing my experience installing it.

## First Things first

I am a linux user, I've tried Ubuntu and Fedora and was using Fedora before switching to Arch.
I know that Arch Linux has a lot features, that makes you want to switch to it, but none of those
features actually did mean anything to me. I just wanted a computer that works, until I wanted 
more.

The reason I switched to using Linux in the first place was performance, I wanted to move fast, and things
needed to look and feel snapy. Don't get me wrong installing almost any linux distros will give you a massive
boot in speed it's just that I have a really bad computer, and I wanted more speed. So I decided to switch to
linux, because that distro was came in barebones.

The problem is however that you need to know how your computer works to install Arch.

## Past Experience

I have used Ubuntu in a past computer I had, and figured to install it in my current one as well.
The problem was that I had only 1 flashdrive with more that 8 Gigs of storage and it broke for some reason.
The only other one I had was one with 4 Gigs. The Ubuntu iso file is like 4.4 Gigs or something, and I need
to make the installation with one larger than the iso file. So, I looked for other distros, so that's when
I stubled on Arch.

I've tried to install Arch Linux before. I followed a youtube tutorial on installing Arch, and typed every command
that was mentioned I did it word for word. I didn't know what I was doing, but after a while I eventually
installed Arch. The problem was however, I coudn't get audio. I don't know what the issue was, it might be
a driver issue or maybe some software I didn't install, I didn't know what I was doing so I uninstalled Arch Linux
and installed Fedora instead.

Fedora worked great, for everything I did it was amaizing, it was pretty smooth and had a great experience.
Now, at some point though I felt I needed more performance. The main reson for this was that, the dnf package
manager commands worked really slow in my computer, and I am pretty sure that it is a problem with my computer.
There were also some other very minor performance issues, mind you I'm not just using this computer for
browsing the internet, but also for development.

## Re-installing Arch Btw

I wanted to install Arch properly this time, I wanted to know how things work. That's when I funbled upon a
series of Youtube videos by Typecraft, it was called Linux for Newbs. It was a game changer, I thought I
didn't wanted to know about how I my computer works, and still get to install Arch and get the performance
boot. So, I followed the tutorials and I have to say I learned a couple of things while dowloading Arch.
I was very spectical wheather I will fail again, to install Arch, but Typecraft actuall explains most of
the stuff I'm doing, so I was able to understand a lot of the stuff I was doing.
Now, I have installed Arch Linux and I just wanted to say installing Arch is not that difficult and you
will get to know about how your computer works.

## Experience using Arch

Just as I thought, using Arch had a performance impact. Stuff is faster than usuall.
I thought using Arch would mean me involving with a lot of system stuff a lot, But I was wrong.
After installing gnome as my display manager things feel pretty much the same as I used fedora.
If I'm honest if you already use linux, I really sure that switching to a pre-installed arch linux computer
would not be that difficult.

You don't have to know how your computer works, you only get to learn more of it.
